# Obsidian 使用指南
_最后更新：2026-03-20_

本文档说明如何使用 Obsidian 格式编写记忆库文件，以及如何配合 memory_loader 进行记忆检索。

---

## 1. Frontmatter（文件属性）

每个记忆库文件建议在开头添加 Frontmatter：

```markdown
---
type: note           # 类型: note | project | daily | theme
updated: 2026-03-20  # 最后更新日期
tags: [工作, 项目A]   # 标签，方便分类
---

# 文件标题

内容...
```

**常用属性**：
| 属性 | 说明 | 示例 |
|------|------|------|
| `type` | 文件类型 | `note`, `project`, `daily`, `theme` |
| `updated` | 更新日期 | `2026-03-20` |
| `tags` | 标签列表 | `[工作, 学习]` |
| `status` | 状态 | `active`, `archived`, `completed` |

---

## 2. 双向链接 [[ ]]

Obsidian 的核心特性，用于关联不同笔记。

### 基本用法

```markdown
今天完成了 [[项目A]] 的需求分析。

参考 [[记忆库/L2_知识库/参考文档/文件路径规范]] 的说明。
```

### 链接别名

```markdown
查看 [[项目A|昨天的项目]] 进展。
# 显示为：查看 昨天的项目 进展。
```

### 链接到标题

```markdown
[[项目A#需求分析]]
[[项目A#需求分析|具体需求]]
```

---

## 3. 嵌入笔记 ![[ ]]

将其他笔记的内容嵌入到当前笔记中。

```markdown
# 嵌入整个笔记
![[项目A]]

# 嵌入特定标题
![[项目A#需求分析]]

# 嵌入特定段落（使用块 ID）
![[项目A#^block-id]]
```

---

## 4. 标签 #tag

用于分类和检索。

```markdown
# 单层级标签
这是关于 #工作 的记录。

# 多层级标签
这是 #项目/重要 的内容。

# 多个标签
涉及 #工作 #学习 两个方面。
```

---

## 5. Callout（标注块）

用于突出显示重要信息。

```markdown
> [!note] 提示
> 这是一个提示信息。

> [!warning] 警告
> 这是一个警告信息。

> [!tip] 技巧
> 这是一个实用技巧。

> [!important] 重要
> 这是重要信息。
```

## 6. 与 memory_loader 配合

### 渐进式加载策略

memory_loader 会识别 Obsidian 双链，预加载关联笔记：

```markdown
用户在问："项目A进展如何？"

1. 读取 记忆库/L1_长期记忆/02-项目/项目A.md
2. 发现文中包含 [[项目B]]、[[技术方案]] 等链接
3. 预加载这些关联笔记，提供更完整上下文
```

### 优化记忆检索的建议

1. **使用有意义的文件名**
   ```markdown
   # 好
   [[用户画像分析]]

   # 不好
   [[2024-03-20]]
   ```

2. **在相关笔记间建立链接**
   ```markdown
   # 在项目A中链接相关技术文档
   使用了 [[微服务架构]] 方案。
   ```

3. **使用标签分类**
   ```markdown
   # 工作相关
   #工作 #项目A

   # 学习相关
   #学习 #英语
   ```

4. **定期更新 Frontmatter**
   ```markdown
   ---
   updated: 2026-03-20
   status: active
   ---
   ```

## 7. 记忆库最佳实践

### L0 工作区

- 保持简洁，只记录当天
- 使用待办列表 `- [ ] 任务`
- 完成后及时归档到 L1

### L1 长期记忆

- 按项目/日期组织
- 使用双链关联相关项目
- 定期回顾和整理

### L2 知识库

- 写成可复用的参考文档
- 使用 Callout 突出关键点
- 建立知识间的链接网络

### L3 用户档案

- 记录价值观、目标、偏好
- 不频繁修改，保持稳定性
- AI 读取但不自动修改

## 8. 示例文件

### 项目笔记示例

```markdown
---
type: project
updated: 2026-03-20
status: active
tags: [开发, 后端]
---

# Agent 系统

## 目标
构建个人 AI 助手系统。

## 进展
- [x] 设计架构
- [x] 实现核心功能
- [ ] 编写文档

## 相关笔记
- [[技术方案]]
- [[Claude Code 配置]]

## 日志

### 2026-03-20
完成了基础框架。
```

### 每日日志示例

```markdown
---
type: daily
updated: 2026-03-20
---

# 2026-03-20

## 上午
- 完成 [[Agent 系统]] 的架构设计
- 整理了需求文档

## 下午
- 开始编码实现
- 遇到 [[技术方案#问题]]

## 明日计划
- [ ] 继续开发
- [ ] 编写测试
```
