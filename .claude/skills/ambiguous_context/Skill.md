# 模糊信息处理 Skill

当用户表达模糊时，主动确认意图，避免误判。

## 触发场景

- 用户说"那个项目"、"这个任务"
- 关键词匹配多个文件
- 时间范围不明确（"最近"、"之前"）
- 操作意图不清楚（"整理一下"、"同步一下"）

## 执行步骤

1. **识别模糊类型**：项目/文件/时间/操作
2. **检查上下文线索**：最近提到的项目、操作过的文件
3. **列出可能选项**（最多3个）：
   - 优先列出最近提到的项目
   - 选项描述要具体，说明内容/用途
4. **询问用户确认**

## 输出格式

```
你是指：
1. [选项1] - [简要说明]
2. [选项2] - [简要说明]
3. [选项3] - [简要说明]

是哪一个？（回复数字或直接说名称）
```

## 示例

### 示例1：项目名称模糊

**输入：** "那个项目进展怎么样？"

**处理：**
- 分析上下文：刚才讨论了AI电商图，昨天提到了AI转译配音
- 列出选项

**输出：**
```
你是指：
1. AI电商图项目（刚才讨论的）
2. AI转译配音项目（昨天提到的）

是哪一个？
```

### 示例2：文件名模糊

**输入：** "看一下日志"

**处理：**
- "日志"可能匹配：L0/scratchpad, L1/Daily_Logs, L1/Projects

**输出：**
```
你想看哪个日志？
1. 今天的工作记录（L0 scratchpad）
2. 最近几天的日志（L1 Daily_Logs）
3. 项目进展日志（L1 Projects）
```

### 示例3：时间范围模糊

**输入：** "最近的情况怎么样？"

**输出：**
```
你想了解哪个时间范围？
1. 今天（从09:22到现在）
2. 最近3天（1月4日-6日）
3. 本周整体（从周一到现在）
```

## 何时跳过

**直接执行，不需要询问：**
- ✅ 用户明确说出项目全名（"AI电商图项目"）
- ✅ 唯一匹配，无歧义
- ✅ 从上下文可以100%确定意图
- ✅ 用户使用标准命令（"Status"、"Archive"）

## 注意事项

- 优先列出最近提到的项目
- 选项描述要具体，说明内容/用途
- 保持对话自然，不要机械罗列
- 如果用户回复"都看看"，依次读取并总结
- 如果用户说"你自己判断"，选择最可能的选项并说明理由
