# 心智模型审查（Munger Observer）

基于查理·芒格的心智模型，审查你的决策和思考过程，发现盲点和认知陷阱。

## 触发方式

**手动触发：**
- "审查一下我的决策"
- "检查今天有什么盲点"
- "用Munger模型分析一下"
- "二阶思考：XXX"
- "心智模型审查"

**日复盘时可选触发：**
- 日复盘后询问："需要用心智模型审查今天的决策吗？"

## 7个心智模型

### 1. 逆向思维（Inversion）
**核心问题：** 什么会出错？反着想

> "告诉我我会死在哪里，这样我就永远不去那里。" —— 芒格

**检查点：**
- 这件事失败的可能是什么？
- 我在假设一切顺利吗？
- 如果结果相反，是因为什么？

### 2. 二阶思维（Second-Order Thinking）
**核心问题：** 后果的后果是什么？

**检查点：**
- 短期好，但长期可能有问题吗？
- 这个决定会引发什么连锁反应？
- 我在用一阶思考吗（只看直接后果）？

### 3. 激励分析（Incentive Analysis）
**核心问题：** 什么行为被奖励？

> "告诉我激励，我就能告诉你结果。" —— 芒格

**检查点：**
- 这个系统中什么行为被奖励？
- 我有没有被错误激励驱动？
- 对方的激励是什么？和我们一致吗？

### 4. 机会成本（Opportunity Cost）
**核心问题：** 没做什么？

**检查点：**
- 为了做这件事，我放弃了什么？
- 有更好的替代方案吗？
- 我在"对的事情"上花时间，还是只在"容易的事情"上？

### 5. 偏差检测（Bias Detection）
**核心问题：** 我的思考有偏差吗？

**常见偏差：**
| 偏差 | 表现 |
|------|------|
| **确认偏差** | 只找支持自己观点的信息 |
| **沉没成本** | 因为已经投入所以继续 |
| **从众效应** | 因为别人都这样做 |
| **可得性偏差** | 高估最近/印象深刻的信息 |
| **锚定效应** | 被第一个信息过度影响 |
| **幸存者偏差** | 只看成功者，忽略失败者 |

### 6. 能力圈（Circle of Competence）
**核心问题：** 我在已知领域内吗？

**检查点：**
- 这件事我真正懂吗？
- 我是不是在假装懂？
- 如果在圈外，我有保持谦逊/谨慎吗？

### 7. 安全边际（Margin of Safety）
**核心问题：** 有缓冲吗？

**检查点：**
- 如果出错了，我有余量吗？
- 我是不是在极限边缘操作？
- 哪里可以加个缓冲？

## 审查流程

### Step 1: 收集今日决策
从 `scratchpad.md` 提取：
- 今日决策
- 核心洞察中发现的模式
- 卡点和未闭环事项（可能包含决策）

### Step 2: 应用心智模型
用7个模型逐个检查，寻找：
- 认知偏差
- 思考盲点
- 二阶后果
- 隐藏的激励

### Step 3: 生成输出
**如果有洞察：** 1-3条简洁的芒格风格观察
**如果没有：** "暂无明显认知陷阱"

## 输出格式

```markdown
## 🧠 心智模型审查

**[模型名称]：** [观察 + 建议]

（如有更多洞察，继续添加）

---

💡 "反过来想，总是反过来想。" —— 雅可比
```

## 输出示例

```
## 🧠 心智模型审查

**机会成本提醒：** 今天花3小时打磨日报系统，但运营多维表格优化（任务#1）被推迟。考虑：工具优化 vs 业务推进，哪个更关键？

**二阶检查：** 游戏移到早上当奖励 —— 一阶：减少晚上失控。二阶：会不会早上也失控？需要观察早上"自己时间"的实际执行效果。

**沉没成本警告：** record skill 触发问题已多次修复未果。考虑：是继续修补，还是换个思路？不要因为已经投入时间就死守。

---

💡 "反过来想，总是反过来想。"
```

## 与日复盘的区别

| 维度 | 日复盘 | 心智模型审查 |
|------|--------|-------------|
| **关注点** | 发生了什么、有什么洞察 | 决策质量、思考盲点 |
| **时间维度** | 今天/本周 | 特定决策 |
| **输出** | 日志、趋势、模式 | 认知陷阱提醒 |
| **频率** | 每天/每周 | 按需触发 |

## 使用场景

1. **做重要决策前** —— "帮我用Munger模型看看这个决定"
2. **日复盘时** —— 作为可选的深度审查
3. **感觉卡住时** —— "我是不是陷入了什么思维陷阱？"
4. **事后反思** —— "今天有什么决策可能有问题？"

## 注意事项

- 不是每次都需要审查 —— 只针对重要/不确定的决策
- 保持简洁 —— 1-3条洞察即可，不要长篇大论
- 客观中立 —— 指出问题，但也认可合理的决策
- 关注模式 —— 如果某种偏差反复出现，记录到 L3
