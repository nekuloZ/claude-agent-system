## 自动触发检测（v3.4 新增）

本 Skill 支持智能场景路由自动检测，作为 **assistant（助手场景）** 的核心组成部分，处理最高频的日常待办场景。

### 检测逻辑

启动时检查 `.claude/.pending_signal.json`：

1. **文件存在且 timestamp 在 5 分钟内？**
2. **scene 匹配 "assistant" 且包含动作相关 keywords？**
3. **符合条件 → 按 scene-router 配置预加载记忆**
4. **清理 signal 文件，进入正常执行流程**

### 助手场景信号示例

```json
{
  "scene": "assistant",
  "confidence": 0.72,
  "keywords": ["要做", "待办", "记得"],
  "triggerText": "要做数据清洗，记得备份"
}
```

### 预加载内容

当检测到助手场景信号时，自动预加载：
- `scratchpad.md`（今日工作记录）
- `next_actions_kanban.md`（当前待办清单 - **关键**）
- `.current_task.json`（当前激活任务）
- `.active_project.json`（项目上下文）
- `habit_data.json`（习惯数据）
- `L3_User/01-Identity/`（用户偏好）

### 场景感知的能力增强

预加载后，action_manager 可以：

#### 1. 智能上下文标记
```
用户: "要做数据备份"
    ↓ 预加载发现当前激活任务: [财务::数据清洗]
    ↓ 自动标记: "- [ ] **要做数据备份** [财务::数据清洗]"
```

#### 2. 重复检测
```
用户: "要做Seedance测试"
    ↓ 预加载发现看板中已有: "- [ ] **做Seedance测试** [AI工具]"
    ↓ 提示: "看板中已有类似待办，是否重复添加？"
```

#### 3. 今日待办统计
```
用户: "今天做什么"
    ↓ 预加载后可直接回复:
    "今天你有 3 个待办：
     1. [财务::数据清洗] 整理报表
     2. [AI工具] 测试Seedance
     3. 买咖啡豆（习惯提醒）"
```

#### 4. 智能列选择
```
用户: "要做修复bug"
    ↓ 预加载发现当前激活任务: [Jarvis::bug修复]
    ↓ 自动放入"进行中"列（而非默认"待办"）
```

### 触发示例流程

```
用户: "要做数据清洗，别忘了备份原始数据"
    ↓ Signal Hook 检测到 assistant 场景 (confidence: 0.78)
    ↓ 关键词匹配: ["要做", "别忘了"]
    ↓ 写入 .pending_signal.json
    ↓ action_manager 启动
    ↓ 预加载看板、当前任务、项目上下文
    ↓ 检测到当前任务 [财务::数据清洗]
    ↓ 智能拆分:
       - [ ] **要做数据清洗** [财务::数据清洗]
       - [ ] **备份原始数据** [财务::数据清洗]
    ↓ 自动写入看板，无需确认
```

## 执行流程

### 创建动作（即时模式）

**核心原则：立即写入看板，不询问确认**

```
用户说"要做XXX" → 立即读取上下文 → 写入看板 → 回复确认（1句话）
```

**步骤：**

1. **必须读取上下文**（失败后继续，标记为空）：
   - `.active_project.json` → 项目缩写
   - `.current_task.json` → 任务缩写
2. **确定目标列**（见下表）
3. **写入看板**：`记忆库/L0_工作区/next_actions_kanban.md`
4. **即时回复**："已添加到[今日待办/待办]：**动作名** [项目::任务]"

**列选择规则：**

| 场景 | 目标列 | 示例 |
|------|--------|------|
| 有活跃任务 (`.current_task.json` 存在) | **进行中** | 当前正专注的任务 |
| 用户明确说"今天" | **今日待办** | "今天要修复bug" |
| 其他情况 | **待办** | 一般性待办 |

### 完成动作

1. 检测当前上下文
2. 在看板中搜索匹配项（优先匹配当前任务上下文）
3. 标记为完成：`- [x] **动作** [上下文] ✅ YYYY-MM-DD`
4. 回复："已标记完成：**动作名**"

## 快捷检查

执行前确认：
1. **读取 `.current_task.json` 了吗？**（决定放入"进行中"还是其他列）
2. **动作格式正确吗？** `- [ ] **动作描述** [项目::任务]`
3. **立即写入，没有询问确认？**（除非动作描述非常模糊）
4. **回复一句话确认？**（"已添加到今日待办：XXX [项目::任务]"）

## 禁止

- ❌ **不要询问确认** - "要添加吗？" 这种对话打断流程
- ❌ **不要罗列待办清单** - 用户没说"查看"就不要展示全部
- ❌ **不要猜测上下文** - 读取失败就留空 `[]`，不要瞎编
